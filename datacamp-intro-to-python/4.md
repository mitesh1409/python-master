### #4.1 What is a function?

A function is a piece of reusable code, that is responsible to do a particular task.

We have many built-in functions in Python, and if required we can write our own custom function as well.

Examples of some of the built-in functions in Python are:  

* `type()`
* `id()`
* `len()`
* `max()`
* `min()`
* `round()`
* `help()`
* `dir()`
etc.

Built-in Functions:  
These are pre-defined functions in Python that you can use without writing your own code.

Imagine a function as a "Black Box" that takes input and produces output.  
Some magic happens inside that "Black Box", but as a consumer we don't need to worry about it.  

As a consumer we don't need to worry about how the function works internally,  
we just need to know about what input it needs and what will be the output,  
so that we can use the function correctly.

### #4.2 How to get help on a function?

You can check Python's documentation for a list of available built-in functions.  

Reference:  
[Python | Built-in Functions](https://docs.python.org/3/library/functions.html)

Or you can use `help()` function.

```python
help(max)

# OR

?max

# OR

max.__doc__   # prints the docstring directly — shorter than help()
```

### #4.3 Type casting functions

* `str()`
* `bool()`
* `int()`
* `float()`

`str()`  
Return a str version of object.

`bool()`
Return a Boolean value, i.e. one of True or False. 

`int()`
Return an integer object constructed from a number or a string, or return 0 if no arguments are given.

`float()`
Return a floating-point number constructed from a number or a string.

### #4.4 Methods

What?

Examples - list methods, str methods

Optional input parameters

Type hinting input parameters and return value


Everything in Python is an Object - means that all data — from simple numbers to complex functions and modules — is represented as an object in memory.

In Python, methods are indeed functions that are "associated" with an object or a class.  
While they look like regular functions, they are defined inside a class and are designed to define the behaviors and actions that an object of that class can perform.

Method = Function that belong to an object or a class

list methods  

```python
fruits = ['Apple', 'Banana', 'Watermelon', 'Papaya', 'Grapes']

print(fruits.index('Papaya'))

print(fruits.count('Apple'))

fruits.append('Mango')

```

str methods  

```python

first_name = 'john'
last_name = 'wick'
full_name = first_name + ' ' + last_name

full_name.capitalize()

full_name.replace('john', 'peter')

```

* Everything = Object
* Different objects have different methods available
* Methods may or may not change the underlying object on which they are called

Object has one or more methods.

### #4.5 Documenting functions or methods

Writing **docstrings** using triple quotes `"""` is the industry standard.  
There are a few formats — the most widely used is **Google Style**:

```python
def calculate_bmi(weight_kg: float, height_m: float) -> float:
    """Calculate Body Mass Index (BMI) for a given weight and height.

    Args:
        weight_kg (float): Weight of the person in kilograms.
        height_m (float): Height of the person in meters.

    Returns:
        float: The calculated BMI value.

    Raises:
        ValueError: If weight or height is zero or negative.

    Examples:
        >>> calculate_bmi(70, 1.75)
        22.86
    """
    if weight_kg <= 0 or height_m <= 0:
        raise ValueError("Weight and height must be positive values")
    return round(weight_kg / (height_m ** 2), 2)
```

**Sections explained:**

| Section | Purpose |
|---|---|
| First line | Short one-line summary |
| `Args` | Each parameter, its type and meaning |
| `Returns` | What the function returns |
| `Raises` | Exceptions the function can raise |
| `Examples` | Usage examples |

**After documenting, `help()` will show your docstring:**

```python
help(calculate_bmi)
# Calculate Body Mass Index (BMI) for a given weight and height.
# Args:
#     weight_kg (float): ...
```

**Other popular docstring formats:**

```python
# NumPy Style — popular in data science
def fn(x):
    """
    Parameters
    ----------
    x : int
        Description of x

    Returns
    -------
    int
        Description of return value
    """

# reStructuredText (reST) Style — used by Sphinx documentation tool
def fn(x):
    """
    :param x: Description of x
    :type x: int
    :returns: Description of return value
    :rtype: int
    """
```

> 💡 **Google Style** is the most readable and widely adopted in modern Python projects. NumPy style is common in data science libraries. Stick with Google Style unless your project/team specifies otherwise.

### #4.6 Type Hinting OR Type Annotations

You can specify types to parameters and return type.  

**Built-in types — fully supported:**

```python
def greet(name: str, age: int, score: float, active: bool) -> str:
    ...
```

**Custom types — fully supported:**

```python
class Employee:
    def __init__(self, name: str, salary: float):
        self.name = name
        self.salary = salary

def give_raise(employee: Employee, amount: float) -> Employee:  # ✅ custom type
    employee.salary += amount
    return employee
```

**Collection types:**

```python
from typing import Any

def process(
    names: list[str],
    scores: dict[str, float],
    data: tuple[int, str],
    anything: Any
) -> list[float]:
    ...
```

**Benefits of type hinting:**

1. **Static analysis** — tools like `mypy` and Pylance catch type errors *before* running code
```python
calculate_bmi("70", 1.75)  # ❌ Pylance warns: expected float, got str
```

2. **IDE autocomplete** — as you experienced earlier with `pd.Series`, IDE knows what methods are available

3. **Self-documenting code** — type hints make code readable without needing extra comments

4. **Easier refactoring** — when you change a type, IDE immediately highlights all affected places

**Important to remember:**

> 💡 Python does **not enforce** type hints at runtime — it's purely for developer tooling and readability. If you pass a wrong type, Python won't throw an error unless you use a tool like `mypy` or a library like `pydantic` for runtime enforcement.

```python
# Python won't stop this at runtime even with type hints
calculate_bmi("hello", "world")  # runs without error at runtime
                                 # but Pylance/mypy will warn you ✅
```
