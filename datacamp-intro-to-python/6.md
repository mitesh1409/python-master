### #6.1 What, why and how of NumPy

The fundamental package for scientific computing with Python

[NumPy](https://numpy.org/)

Why NumPy?

* NumPy provides a variety of mathematical operations over collections.  
* NumPy arrays are fast compared to Python lists.  
* Allows calculations over entire arrays.  
* Less code and more functionality compared to Python Lists.  
* NumPy Array is a better alternative to Python List.  

How NumPy?

Install numpy package and import in your script to use it.

`pip install numpy`

OR

`uv add numpy`


Example:  
Suppose we have a data of heights and weights of the 1000 people and we want to calculate BMI (Body Mass Index) of each person.

Formula to calculate BMI = Weight / (Height) with power of 2

Using Python Lists

```python
# Weights in Kilograms
weights = [65.4, 59.2, 63.6, 88.4, 68.7]

# Heights in Meters
heights = [1.73, 1.68, 1.71, 1.89, 1.79]

bmis = []
for w, h in zip(weights, heights):
    bmis.append(round(w/h**2, 2))

print(bmis)
```

Using NumPy

```python
import numpy as np

# Weights in Kilograms
weights = np.array([65.4, 59.2, 63.6, 88.4, 68.7])

# Heights in Meters
heights = np.array([1.73, 1.68, 1.71, 1.89, 1.79])

bmis = weights / heights ** 2

print(weights)
print(heights)
print(bmis)

print(type(bmis)) # <class 'numpy.ndarray'>
```

### #6.2 NumPy Remarks

* NumPy array elements are stored in the contiguous memory locations.
* NumPy arrays should contain only one type, means it can be an array of floats, array of booleans, array of strings etc.
* When you create NumPy array of mixed type values, all values are converted into string type
* NumPy array is simply a new kind of Python type, like float, int, string etc.
* NumPy array and Python List are two totally different types, they behave differently even if we use the same operand (+, -, * etc.)
* Type Consistency: NumPy arrays require all elements to be of the same type, converting different types to a common one if necessary.
* Array Arithmetic: Arithmetic operations like +, -, *, and / behave differently with NumPy arrays compared to Python lists.
* NumPy array type is `<class 'numpy.ndarray'>` - means ndarray (n-dimensional array) type defined in the numpy package.


We can easily calculate BMI using NumPy arrays,  
using Python lists it is not that straight forward we have to write more code.

```python
# Using NumPy
import numpy as np

# Weights in Kilograms
np_weights = np.array([65.4, 59.2, 63.6, 88.4, 68.7])

# Heights in Meters
np_heights = np.array([1.73, 1.68, 1.71, 1.89, 1.79])

bmis = np_weights / np_heights ** 2

print(bmis)

# Using Python List
# Weights in Kilograms
weights = [65.4, 59.2, 63.6, 88.4, 68.7]

# Heights in Meters
heights = [1.73, 1.68, 1.71, 1.89, 1.79]

# Following is not possible with Python lists
bmis = weights / heights ** 2
```

### #6.4 n-dimensional arrays in NumPy

```python
import numpy as np

# Heights in Meters
heights = np.array([1.73, 1.68, 1.71, 1.89, 1.79])

# Weights in Kilograms
weights = np.array([65.4, 59.2, 63.6, 88.4, 68.7])

array_2d = np.array([
    heights,
    weights
])

print(array_2d)
print(array_2d.shape) # (2, 5) - 2 rows, 5 columns

# Get height and weight of the 2nd and 3rd member.
array_2d[:, 1:3]
# OR
array_2d[:, [1, 2]]

# Get height of all the members, 1st row
array_2d[0, :]
# OR
array_2d[0]

# Get weight of all the members, 2nd row
array_2d[1, :]
# OR
array_2d[1]

# Get height and weight of the 1st member, 1st column
array_2d[:, 0]
```

NumPy: Basic Statistics

```python
import numpy as np

# Suppose we have heights and weights of all the people living in the city.
city_people = np.array([
    [1.64, 71.78], # height in meters, weight in kgs
```

NumPy: Generate Sample Data

```python
import numpy as np

# Generate 5000 records for heights
heights = np.round(np.random.normal(1.75, 0.2, 5000), 2)
# Generate 5000 records for weights
weights = np.round(np.random.normal(60.32, 15, 5000), 2)

# Generate an np array containing heights and weights as its columns.
city_people = np.column_stack((heights, weights))
```

`np.random.normal()` takes 3 arguments:  
* distribution mean
* distribution standard deviation
* number of samples

## Summary

Python lists are flexible but slow for element-wise calculations.

NumPy arrays allow fast, efficient operations over entire collections of data.

You can convert lists to NumPy arrays using np.array().

NumPy arrays contain a single data type - meaning they are homogeneous.
When you provide mixed type values then it will convert all the values to a common type.

NumPy arrays support element-wise calculations.

You can subset arrays using boolean conditions, like bmi > 23.

NumPy great for doing vector arithmetic.
